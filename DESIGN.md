# Yode Design Context

## 设计方向

Yode 桌面端是产品型工作台，不是品牌展示页。界面服务于长时间代码任务：低噪声、结构清晰、状态可检查、关键风险醒目。默认场景是开发者在 macOS 桌面环境中处理代码仓库，窗口常驻在屏幕一侧或全屏使用，因此第一版采用深色工作台。

## 视觉原则

- 信息架构优先于装饰。主区域展示 timeline，左侧承载导航，顶部承载当前会话和 runtime 状态，底部承载输入。
- 卡片只用于结构化事件：工具调用、文件修改、权限确认、设置分组、错误状态。
- 不使用营销式 hero、渐变文字、大面积装饰图形或无意义动效。
- 使用系统字体，保证桌面应用的原生感和可读性。
- 动效只用于状态反馈、展开折叠和 hover/focus，时间控制在 150 到 220ms。

## 初始颜色策略

策略：Restrained。

- 主背景：接近黑色但带轻微冷色调，避免纯黑。
- 面板背景：比主背景稍亮，用于 sidebar、composer、设置分组。
- 边框：低对比冷灰，用于分隔而不是装饰。
- 文本：高亮文本不使用纯白，次级文本保持足够对比。
- Accent：先使用偏玫红的 Yode 临时强调色，仅用于当前选择、主按钮、运行状态和确认动作。
- 语义色：错误、警告、成功、信息必须独立于 accent。

## 初始组件规范

- Sidebar：固定宽度，支持项目、会话、设置入口。当前项使用背景和细边框提示，不使用粗侧边色条。
- Topbar：紧凑显示会话标题、工作区、provider/model、权限模式、运行状态。
- Timeline：append-only event log。用户消息、assistant 回复、reasoning、tool、permission、compact boundary 是不同节点。
- Tool card：默认折叠摘要，展开后显示参数、进度、结果、错误和输出预览。
- Composer：多行输入，附加 controls 放在同一工具条中，包括附件、权限模式、provider/model 和发送/停止。
- Settings：左侧 tab，右侧表单或空态；第一批只实现 shell 和常规页。

## 文案规则

- 所有用户可见中文采用简体中文。
- 不写教学式解释，不在界面中重复说明显而易见的功能。
- 技术名词保持英文，例如 provider、model、MCP、hooks、runtime、diff。
- 状态文案短而具体，例如“等待确认”“工具运行中”“上下文已压缩”。
